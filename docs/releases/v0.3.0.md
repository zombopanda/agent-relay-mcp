# Agent Crossbar v0.3.0

A `profile_health` payload-size fix: model discovery output for large
catalogs (e.g. OpenCode's 60+ live models) is now compacted instead of
spelling out effort names and duplicating model ids.

## Changed

- Effort names (`low`, `medium`, `high`, `max`, `xhigh`, `ultra`) are replaced
  with small int codes in `model_info`, resolved via a top-level
  `effort_legend` field.
- The redundant flat `models[]` list is dropped — `model_info` is now the
  single source of model ids.
- `model_info` entries are grouped by their `provider/` id prefix (e.g.
  `opencode-go`, `google-vertex`), with the prefix stripped from each entry
  id instead of repeated on every model.
- Cache records with a populated `models` list but empty/missing
  `model_info` (older or partial cache entries) fall back to bare
  `{"id": ...}` entries instead of silently losing their model ids.

Net effect: a `profile_health` response for a catalog with hundreds of
models now serializes at roughly a tenth of its previous size, with no loss
of information needed to select a model and effort level.
