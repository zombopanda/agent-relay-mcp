# Agent Crossbar v0.2.0

The first public release of Agent Crossbar: one small MCP surface for asking,
delegating to, and reviewing with locally installed Codex, Claude Code,
OpenCode, and experimental Reasonix harnesses.

This release replaces the rejected `agent-relay-mcp` identity. The accidentally
published `agent-relay-mcp==0.1.3` package is yanked with a pointer to
`agent-crossbar`; it is not deleted.

## Qualified surface

| Profile | Ask | Dev | Review | Interactive runtime |
|---|---:|---:|---:|---|
| Codex | Yes | Yes | Yes | tmux |
| Claude Code | Yes | Yes | Yes | background/tmux |
| OpenCode | Yes | Yes | Yes | ACP/tmux |
| Reasonix | Experimental | No | No | print |

The release candidate passed:

- 877 Python tests with 11 provider-dependent skips;
- Ruff lint and formatting checks;
- clean-runner CI on Python 3.11, 3.12, and 3.13 on Ubuntu plus Python 3.13
  on macOS;
- wheel, sdist, npm tarball, secret, dependency, CodeQL, and artifact audits;
- live ask/dev/review provider gates for the default Codex, Claude Code, and
  OpenCode profiles;
- a live Reasonix ask gate;

## Safety and billing

Claude print transport is disabled until the user explicitly consents because
its billing path can be ambiguous. Existing Anthropic API-key use is treated as
explicit API-billing intent. Interactive Claude sessions use the configured
Claude Code OAuth runtime.

Provider failures return structured readiness, timeout, authentication, and
rate-limit feedback. `agent-crossbar doctor` checks the installed harnesses
before the first job.

## Known limits

- Reasonix remains experimental and ask-only.
- ACP is used where the installed harness exposes a compatible agent; tmux is
  retained for interactive harnesses that do not.
- Registry propagation can briefly delay a just-published version; the release
  workflow verifies the exact PyPI artifact before publishing the npm launcher.
